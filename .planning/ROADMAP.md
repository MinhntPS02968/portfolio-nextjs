# Roadmap: Frontend Developer Portfolio

## Overview

Roadmap v1 tập trung đưa portfolio hiện có thành một trải nghiệm recruiter-ready: trước hết hoàn thiện luồng nội dung cốt lõi, sau đó đồng bộ visual neon mobile-first, và cuối cùng tối ưu motion/performance/accessibility để đạt cảm giác mượt, nhanh, dễ đọc và dễ bảo trì.

## Phases

- [ ] **Phase 1: Hero & About Foundation** - Hoàn thiện ấn tượng đầu tiên và định vị cá nhân rõ ràng ngay khi vào trang.
- [ ] **Phase 2: Showcase Content & Update Flow** - Hoàn thiện phần kỹ năng/dự án/kinh nghiệm và đảm bảo cập nhật nội dung dễ dàng.
- [ ] **Phase 3: Neon Visual System & Responsive Readability** - Chuẩn hóa nhận diện neon, độ tương phản và layout mobile-first trên toàn bộ section chính.
- [ ] **Phase 4: Motion Polish, Accessibility & Runtime Performance** - Bổ sung parallax/transition mượt, tôn trọng reduced motion, và giữ tốc độ tải/interaction nhanh.

## Phase Details

### Phase 1: Hero & About Foundation
**Goal**: Người xem hiểu ngay bạn là ai, bạn mang lại giá trị gì và có hướng hành động rõ ràng trong vài giây đầu.
**Depends on**: Nothing (first phase)
**Requirements**: PORT-01, PORT-02
**Success Criteria** (what must be TRUE):
  1. User can see a Hero section with role, short value statement, and a clear primary CTA on first view.
  2. User can read an About section that quickly communicates background and frontend specialization.
  3. User can navigate from Hero to About smoothly on both mobile and desktop layouts.
**Plans**: TBD
**UI hint**: yes

### Phase 2: Showcase Content & Update Flow
**Goal**: Người xem có thể đánh giá năng lực thực tế qua skills, projects và experience; maintainer cập nhật dữ liệu portfolio không cần refactor lớn.
**Depends on**: Phase 1
**Requirements**: PORT-03, PORT-04, PORT-05, PERF-02
**Success Criteria** (what must be TRUE):
  1. User can review skills grouped by relevant frontend technologies in a scannable format.
  2. User can browse project summaries with clear outcomes and context for each project.
  3. User can view experience timeline in chronological order without ambiguity.
  4. Maintainer can update projects, skills, and experience content through a centralized, low-friction data structure.
**Plans**: TBD
**UI hint**: yes

### Phase 3: Neon Visual System & Responsive Readability
**Goal**: Portfolio có nhận diện neon nhất quán, hiện đại, nhưng vẫn dễ đọc và ổn định trên mobile-first breakpoints.
**Depends on**: Phase 2
**Requirements**: VIS-01, VIS-02, VIS-03
**Success Criteria** (what must be TRUE):
  1. User sees a cohesive neon visual identity applied consistently across Hero, About, Skills, Projects, and Experience.
  2. User can read primary text comfortably with strong contrast and legibility despite neon styling.
  3. User can browse all major sections with responsive behavior optimized for small screens first.
**Plans**: TBD
**UI hint**: yes

### Phase 4: Motion Polish, Accessibility & Runtime Performance
**Goal**: Trải nghiệm chuyển động mang cảm giác premium nhưng không gây jank; website nhanh, truy cập được bằng keyboard và hỗ trợ reduced motion.
**Depends on**: Phase 3
**Requirements**: MOT-01, MOT-02, MOT-03, PERF-01, PERF-03
**Success Criteria** (what must be TRUE):
  1. User experiences balanced parallax effects on supported devices without distracting content readability.
  2. User sees smooth section transitions and hover interactions with no obvious jank during normal browsing.
  3. User can navigate core content with keyboard and semantic structure support.
  4. User who prefers reduced motion can still complete key navigation and content consumption comfortably.
  5. User gets fast initial load and responsive interactions on common mobile devices.

**UI hint**: yes
