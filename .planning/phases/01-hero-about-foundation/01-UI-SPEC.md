# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Bootstrap 5.3 utilities + custom BEM-style portfolio components |
| Icon library | Material Symbols Outlined |
| Font | Headings: Space Grotesk 600; Body/UI: Manrope 400, Roboto fallback |

Implementation constraints for this phase:
- Mobile-first layout behavior from 320px upward.
- Accessibility baseline required: keyboard reachable controls, visible focus, readable contrast.
- No inline CSS; styling must live in SCSS/CSS files.
- Keep structure maintainable: reuse existing `pf-*` utility/component patterns.

## Interaction Contract

Hero focal point: Role title and short value statement in the hero text block are the primary visual focus on first view; decorative neon/parallax layers must remain secondary and must not compete with this content.

| Area | Contract |
|------|----------|
| Hero first view | Above-the-fold must show role, short value statement, and primary CTA without extra scroll on common mobile widths. |
| Hero -> About navigation | `#about` anchor scroll must be smooth and stable on mobile/desktop; active nav state updates with section visibility. |
| Neon effects | Use soft glow layers (`opacity <= 0.2`) behind text/content, never over paragraph text. |
| Parallax policy | Keep parallax subtle (slow background delta only), disable/degrade on low-power/reduced-motion contexts, and never block reading flow. |
| Reduced motion | If `prefers-reduced-motion: reduce`, disable parallax and long pulse animations; keep all content and CTA fully usable. |
| Hover/focus behavior | Hover enhancements are optional; keyboard focus styles are mandatory and use high-contrast outline/glow. |
